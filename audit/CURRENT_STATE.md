# Текущее Состояние Проекта (Current State)

Проект находится в стадии **Production-Readiness (Preparation for VPS)**.
Фундамент построен, архитектура утверждена. Запрещены дальнейшие миграции старого кода "ради красоты". Цель — первый тестовый заказ.

## Готово (✅)
1. **EventBus & EventJournal**
   - Событийно-ориентированная нервная система работает.
   - Гарантия доставки событий (EventJournal сохраняет события до их обработки).
2. **Финансы (Ledger)**
   - Профит рассчитывается только `FinanceService`.
   - Защита от двойных начислений (Idempotency check на уровне БД и статуса).
3. **Доставка (Delivery)**
   - Полный флоу протестирован: Payment -> OrderService -> DeliveryService -> CCE -> Ledger.
   - Поддерживает обработку ошибок провайдера (FAIL_DELIVERY).
   - Поддерживает ситуации падения CCE (DELIVERY_PENDING).
4. **CCE (Customer Communication Engine)**
   - Управление шаблонами, стейджами сообщений и автоответами инкапсулировано.
5. **Безопасность (Real Mode Safety)**
   - Sandbox-режим для плагинов (они не могут трогать БД напрямую).

## Требует Внимания (⚠️)
1. **Интеграция с реальным API FunPay**
   - На данный момент симуляция работает отлично, но методы взаимодействия с FunPay в фасаде (`seller_service.py`) должны быть проверены с реальным аккаунтом на мелких суммах.
2. **Интеграция реального поставщика**
   - В `delivery_service.py` заглушка для API поставщика. Нужно вставить реальные HTTP запросы.
3. **Секреты (Env/Secrets)**
   - Не настроен `.env` для production, ключи пока загружаются из локальных конфигов.

## Не Готово (⛔)
- **Мониторинг сервера (Prometheus/Grafana)** - пока не внедрен.
- **Интеграционные тесты с реальным FunPay API**.
