# FunPay Hub - Audit Package

Добро пожаловать в репозиторий FunPay Hub!
Этот проект представляет собой автоматизированную систему для торговли на площадке FunPay с использованием Event-Driven архитектуры, плагинов, симуляции и строгого контроля финансов (Ledger).

## Описание проекта
Проект разделен на "Ядро" (Runtime) и "Плагины". Ядро обеспечивает безопасность, надежность, транзакционность (через EventJournal) и взаимодействие с CCE (Customer Communication Engine). Плагины реализуют конкретные бизнес-сценарии (автовыдача, мониторинг цен, автоподнятие лотов и т.д.).

## Содержимое Audit Package
Этот пакет содержит документацию, необходимую для независимого аудита и запуска первой боевой версии проекта:

1. **[ARCHITECTURE_FREEZE.md](ARCHITECTURE_FREEZE.md)** - Правила заморозки архитектуры и допустимые изменения. **[ОБЯЗАТЕЛЬНО К ПРОЧТЕНИЮ]**
2. **[CURRENT_STATE.md](CURRENT_STATE.md)** - Текущий статус проекта (что готово, что нет).
3. **[ARCHITECTURE.md](ARCHITECTURE.md)** - Описание Control Center 2.0 и Event-Driven структуры проекта.
4. **[SECURITY.md](SECURITY.md)** - Ключевые ограничения безопасности и контроля (Sandbox, Ledger).
5. **[VPS_DEPLOY.md](VPS_DEPLOY.md)** - Шаги для запуска на сервере VPS.
6. **[TEST_RESULTS.md](TEST_RESULTS.md)** - Результаты финальных E2E тестов.
7. **[KNOWN_ISSUES.md](KNOWN_ISSUES.md)** - Известные технические долги.
8. **[DATABASE_SCHEMA.md](DATABASE_SCHEMA.md)** - Архитектура базы данных.
9. **[FIRST_PRODUCTION_CHECKLIST.md](FIRST_PRODUCTION_CHECKLIST.md)** - Чеклист перед первым реальным заказом.

**Цель пакета:** Дать возможность новому ИИ / DevOps инженеру быстро понять проект, оценить его готовность к production и безопасно запустить на VPS без необходимости рефакторить весь код.
