# Known Issues & Technical Debt

Список того, что было осознанно отложено до первого тестового запуска:

1. **`delivery_service.py` API Поставщика**
   На данный момент генерация ключа симулируется строкой `SIMULATION-KEY-{order_id}`. При реальном деплое необходимо заменить этот участок кода на реальный HTTP POST запрос к API поставщика (например, Steam API, сторонние селлеры).

2. **Остаток `seller_service.py`**
   Из 90+ методов в `seller_service.py` часть перенесена в сервисы (Orders, Lots, Chat). Часть всё еще осталась (методы статистики, старой логики профиля). Они не тронуты из-за правила `ARCHITECTURE_FREEZE.md`.

3. **Логирование**
   Логи раскиданы по stdout и нескольким текстовым файлам. Для Production желательно подключить структурированное логгирование (JSON) и собирать их через Loki или ELK, но для первого старта достаточно и `tail -f`.

4. **Мониторинг цен (Price Monitor Plugin)**
   Не адаптирован под новые API `LotService`. Работает в "Monitor Mode" (read-only).
