# FunPayHub Architecture Overview

Эта документация и все связи генерируются автоматически на основе AST-анализа исходного кода проекта.

## Кластеры системы

Архитектура проекта разделена на четко определенные кластеры, каждый из которых отвечает за свою область бизнес-логики.

| Кластер | Назначение |
|---|---|
| **Core Runtime** | Инициализация (`hub_bootstrap.py`, `funpayhub_main.py`), EventBus, управление фоновыми процессами. |
| **CCE (Customer Communication Engine)** | Единый слой общения с покупателями (`messages`, `notifications`). |
| **Order Engine** | Отслеживание жизненного цикла заказов (`order_flow.py`, `order_tracker.py`). |
| **Plugins** | Бизнес-логика, которая может включаться и выключаться на лету (AutoSMM, AutoDonate, Stars). |
| **Database** | Взаимодействие с хранилищем данных и слой финансового журнала (Ledger). |
| **Web API** | RESTful интерфейсы для дашборда и интеграции с ботом (`web/`). |
| **Bot** | Telegram-бот для управления хабом (`bot/`). |
| **AI** | Автоматизированная ИИ-команда для аудита и самовосстановления системы. |
| **FunPayAPI** | Низкоуровневая обертка для работы с API самого маркетплейса. |

## Граф Зависимостей

Полную интерактивную карту с графом зависимостей и списком классов каждого модуля вы можете посмотреть, открыв файл `docs-site/index.html` в вашем браузере.

Карта генерируется на лету из файла `/docs/generated/project_map.json`, который обновляется скриптом `tools/project_analyzer.py`.
