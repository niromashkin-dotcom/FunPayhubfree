# Project Rules: Core Frozen & Architecture Constraints

## 1. Заморозка Ядра (Core Frozen)
Ядро системы (Core) и плагин AutoSMM считаются завершенными и стабильными. Любые изменения в коде ядра **строго запрещены**.

### Запрещенные для изменения директории и файлы:
* `runtime/` (включая `runtime/database/`, `runtime/messages/`, `runtime/order_flow.py`, `runtime/lot_generator.py` и др.)
* `bot/` (включая `bot/runner.py`, `bot/plugins/` и др.)
* `plugins/autosmm_plugin.py`
* `eventbus.py` (если применимо)
* `delivery_service.py` (если применимо)

Если для бизнес-логики потребуется изменить что-либо в ядре, это допускается исключительно после отдельного аудита, обоснования и согласования с пользователем.

---

## 2. Архитектура Control Bot v2
Control Bot v2 должен разрабатываться как **полностью изолированный внешний сервис**.

### Разрешенные директории для разработки:
* `control_bot/` (код бота)
* `deploy/funpayhub-control-bot.service` (юнит автозапуска)
* `scripts/` (вспомогательные скрипты управления)
* `tests/control_bot/` (тесты для нового бота)
* `docs/` (документация)
* `configs/control_bot.json` (конфигурация нового бота)

### Ограничения интеграции:
* **Запрещено импортировать внутренние модули Core** (из `bot.*` или `runtime.*`) в код Control Bot v2.
* Взаимодействие с Core должно осуществляться строго через:
  1. Публичное API ядра (Core API).
  2. SQLite базы данных (только чтение в тех местах, где это допустимо для получения статуса).
  3. Управление через systemd (проверка статуса службы).
  4. Чтение журнала логов ядра (`journalctl` / файлы логов).
