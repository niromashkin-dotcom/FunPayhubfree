# ACCEPTANCE TEST PLAN — FunPayHub

**Дата:** 13.07.2026  
**Статус:** НЕ НАЧАТ — запрет на новый функционал до прохождения всех этапов  
**Правило:** Ни одной новой кнопки, ни одного нового плагина, ни одного нового эндпоинта до полного прохождения Acceptance Test.

---

## Общий принцип

Acceptance Test ≠ поиск багов.  
Acceptance Test = проживание проекта как единого организма.

Проверять не отдельные функции, а **непрерывный цикл** от запуска до отчёта.

Каждый этап завершается одним из вердиктов:
- **PASS** — этап пройден полностью, переходим к следующему
- **PARTIAL** — работает с оговорками, требуется доработка перед следующим этапом
- **FAIL** — критическая проблема, не позволяющая продолжить
